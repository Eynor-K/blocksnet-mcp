# Контракт инструмента `analyze_urban_question`

Основной (и единственный) MCP-инструмент сервера. Запускает полный агент на **локальных данных** и
возвращает строгий машиночитаемый JSON.

---

## 1. Назначение

> Агент-анализ обеспеченности/доступности города и **измеренные предложения развития** (`blocksnet`).
> На вход — задача на естественном языке; внутри — ANALYSIS PLAN → инструменты `blocksnet` →
> PTR-гипотезы → измеренное предложение `before→after`; на выход — структурированный JSON.

## 2. Вход (`params`)

| Параметр | Тип | Обяз. | Описание |
|---|---|---|---|
| `question` | `str` | ✅ | задача на естественном языке (напр. «Где разместить спортивные площадки?») |
| `max_iterations` | `int` | — | лимит итераций агента, по умолчанию `10` |

Данные берутся из локальной модели в `DATA_DIR` (см. [deployment.md](deployment.md)). Вопрос за
пределами модели агент честно помечает в `limitations`.

> 🕘 **Future:** параметры `scenario_id` / `project_id` (контекст UrbanDB) — вне текущего объёма;
> появятся вместе с адаптером данных при выходе на платформу. Сейчас не принимаются.

## 3. Выход (JSON)

```json
{
  "question": "...",
  "analysis_plan": "...",
  "result": "...",
  "hypotheses": [
    { "id": "1", "claim": "...", "prediction": "...", "test": "...",
      "status": "supported|refuted|inconclusive", "evidence": "..." }
  ],
  "measured": {
    "pitch": { "strong_before": 0.732, "strong_after": 0.781, "missing_before": 785, "missing_after": 769 }
  },
  "recommendation_blocks": [2, 4, 5, 11, 12],
  "confidence": 0.78,
  "limitations": ["..."],
  "artifacts": ["maps/provision.png", "scenario.csv"],
  "run_id": "run_2026..."
}
```

| Поле | Источник в `AgentResult` | Назначение |
|---|---|---|
| `question` | `input` | исходная задача |
| `analysis_plan` | секция `ANALYSIS PLAN` | рассуждение **до** расчётов: потребности → гипотезы → метрики/инструменты |
| `result` | секции `RESULT` (+ `REFLECTION`) | ключевые числа модели с единицами и интерпретацией (полный нарратив) |
| `hypotheses[]` | PTR-леджер (`hypotheses.py`) | гипотезы с фальсифицируемым предсказанием и классифицированным исходом |
| `measured` | наблюдения сценария `before→after` | числовой эффект предложения (strong/missing до и после) |
| `recommendation_blocks` | кандидатные кварталы | `block_id` рекомендуемых для развития кварталов |
| `confidence` | `confidence` | авторитетный авто-скоринг 0.0–1.0 (улики + статусы гипотез) |
| `limitations` | `limitations` | что не учтено; явная пометка выхода за пределы локальной модели |
| `artifacts` | пути из `run_dir` | CSV + картограммы (`maps/*.png`) — локальные пути прогона |
| `run_id` | `run_dir` | идентификатор прогона с артефактами |

> Длинный нарратив остаётся в `result` / `analysis_plan`, но **ключевое** (гипотезы, числа эффекта,
> кандидатные кварталы, `confidence`) вынесено в отдельные машиночитаемые поля — чтобы потребитель не
> парсил текст.

## 4. Примеры

**Вход** (генеративный вопрос):
```json
{ "question": "Где разместить новые спортивные площадки?" }
```

**Выход** (сокращённо):
```json
{
  "question": "Где разместить новые спортивные площадки?",
  "analysis_plan": "Потребность: дефицит спорта в кварталах с высокой плотностью населения...",
  "hypotheses": [
    { "id": "1", "claim": "Спорт-дефицит концентрируется на востоке", "prediction": "provision < 0.5 в ≥5 кварталах",
      "test": "compute_service_provision(pitch)", "status": "supported", "evidence": "7 кварталов с provision 0.31–0.48" }
  ],
  "measured": { "pitch": { "strong_before": 0.732, "strong_after": 0.781, "missing_before": 785, "missing_after": 769 } },
  "recommendation_blocks": [2, 4, 5, 11, 12],
  "confidence": 0.78,
  "limitations": ["Учтена только текущая модель города из data/; новые жилые проекты не заложены"],
  "artifacts": ["maps/sport_provision.png", "scenario_before_after.csv"],
  "run_id": "run_20260617_004403"
}
```

## 5. Поведение и ограничения вызова

- **Долгий вызов:** один прогон 60–120 с (с TPE-оптимизацией дольше). Учесть таймаут MCP-клиента и `max_iterations`.
- **Идемпотентность:** в рамках запуска `compute_*` мемоизируются (нет повторных расчётов и дублей артефактов); между запросами кэш сбрасывается.
- **Артефакты:** локальные пути в `outputs/run_*` (на вашей машине).

## 6. Опциональные прямые инструменты (позже, при желании)

Поверх `analyze_urban_question` можно добавить «тонкие» инструменты для дешёвых вызовов без полного
агента — например `compute_service_provision`, `compute_scenario_provision`. Необязательно для локального MVP.
