03 — Ключевое исправление · резолвинг имён сервисов

Найдено в анализе прогона

На вопрос «Где разместить новые спортивные площадки?» оптимизатор измерял не те сервисы: service_set='sports' молча уходил в пресет basic (convenience / kindergarten / school), а целевой pitch выпадал из измеренного сценария.

Фикс (вариант A): имя сервиса определяется по данным каталога (service_type.json: name + name_ru + keywords) и data/service_aliases.json; при неудаче — честная ошибка вместо basic. Хардкод алиасов из кода удалён. Теперь 'sports' → pitch.

Регрессионные тесты

tests/test_service_resolution.py — 12 кейсов:

  • распознавание известных сервисов (рус/англ, name_ru, алиасы);
  • регрессия: service_set='sports' → фокус на pitch, не basic;
  • compute_scenario_provision фильтрует измерение по целевому сервису;
  • явный mismatch: если сценарий без целевого сервиса — отчёт это сообщает;
  • неизвестный service_set → UnknownServiceSet (честная ошибка).
Проверено · регрессия маршрутизации (блокнот + хелперы ядра)

service_set='sports' → {pitch: 1.0} · сценарий {sports:10, convenience:5} нормализуется в {pitch, convenience} · сценарная проверка считает только pitch, convenience исключён. Все ✅ pass.
